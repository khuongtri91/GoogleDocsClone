## 🔄 Development Workflow (MANDATORY)

For every feature, bug fix, or task, follow this process:

### 1. Understand Requirements

* Clarify the goal
* Identify inputs, outputs, constraints
* Ask questions if unclear

### 2. Design

* Propose architecture or approach
* Identify components, data flow, and dependencies

### 3. Plan

* Break work into clear steps
* Define order of execution
* Highlight risks or edge cases

### 4. Implement

* Write clean, maintainable code
* Follow all project rules (React, Tailwind, MUI, performance)

### 5. Review

* Check for:

  * correctness
  * performance issues
  * consistency with project standards

### 6. Test

* Add or suggest test cases
* Consider edge cases and failure scenarios

### 7. Deploy (if applicable)

* Ensure code is production-ready
* Avoid breaking changes

### 8. Monitor (if applicable)

* Suggest logging, metrics, or observability if needed

---

## 🧠 Agent Instructions (CRITICAL)

* ALWAYS follow this workflow unless explicitly told otherwise
* DO NOT jump directly to coding
* Show reasoning step-by-step when handling non-trivial tasks
* If the task is simple, you may compress steps but MUST still consider them internally
